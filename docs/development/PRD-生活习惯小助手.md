## 1. 产品概述

### 1.1 项目背景

现代都市人群普遍面临亚健康状态：久坐少动、作息紊乱、饮食失衡、情绪焦虑。世界卫生组织研究表明，人类健康有 60% 取决于生活方式。中医「治未病」思想与《黄帝内经》"法于阴阳，和于术数，食饮有节，起居有常"的核心理念，正是指向通过日常习惯的调整来预防疾病、养护生命。

然而市面上的习惯类应用存在三大痛点：
- **文化断层**：以西式 GTD/task 理念为主，缺乏中医养生智慧与节气、体质、子午流注等本土化理论支撑；
- **数据孤岛**：习惯打卡与养生知识、AI 咨询相互割裂，用户"记而不知、知而不行"；
- **网络依赖**：多数应用强依赖云端，离线不可用，隐私数据上传第三方。

「生活习惯小助手」正是为解决上述痛点而生：以**离线优先**的 PWA + Android 架构，融合**24 部养生经典著作**与**通义千问 AI**，打造一款"能记录、能学习、能咨询、能提醒"的随身养生伴侣。

### 1.2 产品目标

#### 1.2.1 北极星指标
**月活跃用户平均每周打卡天数 ≥ 5 天**（习惯养成度）。

#### 1.2.2 阶段性目标

| 时间维度 | 目标 | 衡量指标 |
|----------|------|----------|
| 短期（3 个月） | 验证核心打卡闭环，建立种子用户群 | DAU/MAU ≥ 30%；7 日留存 ≥ 40% |
| 中期（6 个月） | AI 顾问与知识库形成差异化壁垒 | AI 月活渗透率 ≥ 50%；人均月咨询 ≥ 8 次 |
| 长期（12 个月） | 建立养生社区与数据资产 | 用户主动导出报告 ≥ 20% 用户/月；Play Store 评分 ≥ 4.5 |

#### 1.2.3 设计原则
1. **离线优先**：核心功能（打卡、统计、提醒、番茄钟）100% 离线可用，AI/推送为增强项；
2. **文化沉浸**：节气、体质、五色五味等中医元素贯穿 UI 与文案，非生硬堆砌；
3. **轻量克制**：APK ≤ 8MB，首屏加载 < 2s，不引入重型框架；
4. **隐私自主**：用户数据仅存本地 localStorage，AI 调用经自部署 Worker 代理，不直连第三方；
5. **双端一致**：PWA 与 Android 同一份 www/ 代码，通过 `__PLATFORM__` 分化能力。

### 1.3 产品范围

#### 1.3.1 范围内（In Scope）
- 习惯打卡（boolean / timer / count / water / select 五类型）
- 养生知识库（二十四节气、子午流注、九种体质、五劳五色、24 部经典摘要）
- AI 养生顾问（通义千问对话，引用典籍）
- 数据统计与可视化（周/月趋势、热力图、等级、成就）
- 提醒系统（轻提醒 / 系统通知 / 强提醒 / 间隔提醒，PWA+APK 双通道）
- 番茄专注钟（25/5/15 标准番茄法）
- 数据导入导出（JSON / CSV）
- 体质测评（9 类型问卷 + 海报分享）
- 新手引导（6 步引导）

#### 1.3.2 范围外（Out of Scope）
- 账号系统与云端同步（v2.0 再议）
- 医疗诊断、处方开具（合规红线）
- 第三方社交平台分享接入（仅本地海报导出）
- iOS 原生应用（暂仅 PWA + Android）
- 付费订阅与商业化

### 1.4 目标用户画像

| 画像 | 养生小白 | 职场亚健康族 | 中医文化爱好者 |
|------|----------|--------------|----------------|
| 年龄 | 20-30 | 25-40 | 30-55 |
| 痛点 | 想养生不知从何做起 | 久坐失眠焦虑，没整块时间 | 有理论基础但难坚持 |
| 核心诉求 | 一键导入习惯包 + AI 答疑 | 番茄钟 + 间隔提醒 + 数据可视化 | 节气养生 + 体质调理 + 经典阅读 |
| 使用场景 | 通勤碎片 | 工作间隙 | 晨起/睡前 |

### 1.5 核心价值主张
> **"以千年养生智慧为体，以现代 AI 技术为用，让每一次打卡都成为一次与身体的对话。"**

## 2. 功能需求

## 3. 验收标准

> 采用 Given-When-Then 格式，每条可测试、可量化。

### 3.1 习惯打卡系统

**AC-1.1 习惯库添加**
- **Given** 用户在「管理」页打开习惯库且尚未添加"晨跑"
- **When** 用户点击"晨跑"项的添加按钮
- **Then** "晨跑"出现在当日打卡列表，`habitsConfig` 新增一条记录，localStorage `habits_config` 持久化，列表 UI 在 200ms 内刷新

**AC-1.2 计时打卡**
- **Given** 用户已添加 timer 类型习惯"晨跑"，当日尚未打卡
- **When** 用户在打卡卡片输入 30 并确认
- **Then** 当日 `checkinRecords[今日][habitId]` 写入 `{done:true, value:30}`，卡片显示"✓ 30 分钟"，streak +1

**AC-1.3 饮水可视化**
- **Given** 习惯"每日饮水"配置 `dailyGoal:2000, perCup:250`，当日已喝 500ml
- **When** 用户点击第 3 个空杯
- **Then** 该杯变为已填充状态（带 ✓），`value` 更新为 750ml，进度百分比从 25% 更新为 38%，剩余杯数从 6 更新为 5

**AC-1.4 负向习惯**
- **Given** 用户添加了 `negative:true` 的"吸烟"习惯
- **When** 用户当日未勾选（即未吸烟）
- **Then** 该日统计视为"达成"，streak +1；卡片视觉与正向习惯区分（如灰色/警示色）

### 3.2 养生知识库

**AC-2.1 节气提示**
- **Given** 系统日期为 2026-02-04（立春）
- **When** 用户打开应用首页
- **Then** 顶部显示"🌱 立春"及提示文案"春季开始，养肝护阳，夜卧早起"

**AC-2.2 离线阅读**
- **Given** 设备处于飞行模式
- **When** 用户进入知识库点击《黄帝内经》
- **Then** 在 1s 内打开 `references/黄帝内经/黄帝内经.html`，无网络请求，内容可正常浏览

**AC-2.3 子午流注**
- **Given** 当前时间为 23:00-01:00（子时）
- **When** 用户查看子午流注图
- **Then** 高亮显示"胆经当令"，并展示"子时养胆宜入睡"相关说明

### 3.3 AI 养生顾问

**AC-3.1 正常问答**
- **Given** 用户已配置有效 Worker URL 且网络可达
- **When** 用户发送"湿气重怎么调理"
- **Then** AI 在 5s 内返回流式回答，字数 ≤ 200，包含"具体方法"分点建议与"引用出处"（如《黄帝内经》），UI 正确渲染 Markdown

**AC-3.2 安全边界**
- **Given** 用户提问"我胃痛该吃什么药"
- **When** AI 处理该请求
- **Then** 回答不包含具体药物处方，并以"⚠️ 以上建议仅供参考，症状持续请及时就医"结尾

**AC-3.3 密钥隔离**
- **Given** 反编译 APK 或抓包 HTTPS 请求
- **When** 检查客户端代码与请求体
- **Then** 不含 `QWEN_API_KEY` 明文，API Key 仅存在于 Cloudflare Worker 环境变量中

**AC-3.4 离线降级**
- **Given** 设备无网络
- **When** 用户尝试发送 AI 提问
- **Then** 提示"网络不可用，AI 顾问需联网"，不崩溃，其他离线功能不受影响

### 3.4 数据统计与可视化

**AC-4.1 周柱状图**
- **Given** 用户本周有打卡数据
- **When** 进入「我的」页面
- **Then** 渲染 7 根柱状图（日一二三四五六），每根高度对应当日完成率百分比，"今天"柱用渐变高亮色，柱高计算公式 `h = max(2, pct/100 * 120)px`

**AC-4.2 连续天数**
- **Given** 用户"早起"习惯已连续打卡 5 天（含今日）
- **When** 查看该习惯卡片
- **Then** 显示"🔥 5 天"streak 标识；同时显示历史最大连续天数

**AC-4.3 等级体系**
- **Given** 用户累计有效打卡 30 天
- **When** 查看个人等级
- **Then** 等级显示为"Lv.5 养生大师 👑"，并解锁对应成就徽章

### 3.5 提醒系统

**AC-5.1 定时提醒（APK）**
- **Given** Android 设备，"晨跑"习惯配置提醒时间 07:00
- **When** 系统时间到达 07:00 且 App 处于后台或被杀
- **Then** 系统通知栏弹出提醒，渠道为 `habit_reminders`，smallIcon 为 `ic_stat_notify`，点击通知打开 App 并定位到打卡页

**AC-5.2 间隔提醒**
- **Given** "起身活动"配置间隔 45 分钟，时段 09:00-18:00，周一至周五
- **When** 工作日 10:30（距上次 09:45 已 45 分钟）
- **Then** 触发一次提醒；当时间为 18:01 或周六时不触发

**AC-5.3 免打扰**
- **Given** 用户设置免打扰时段 22:00-07:00
- **When** 系统时间处于该时段且有提醒计划
- **Then** 该提醒被静默跳过（不注册原生通知），不发出声音与震动

**AC-5.4 PWA 后台推送**
- **Given** PWA 用户已订阅 Web Push 并上传 schedule
- **When** 浏览器关闭且到提醒时间
- **Then** Cloudflare Worker Cron 触发，通过 VAPID + aes128gcm 加密下发推送，设备收到系统级通知

### 3.6 番茄专注钟

**AC-6.1 标准周期**
- **Given** 用户启动番茄钟，模式为 work
- **When** 倒计时从 25:00 走到 00:00
- **Then** 播放完成音效，模式自动切换为 shortBreak（5:00），cycle +1，当日专注统计 +1 轮

**AC-6.2 关联习惯**
- **Given** 用户在番茄钟面板选择关联习惯"阅读"
- **When** 一个工作番茄（25 分钟）完成
- **Then** 当日"阅读"习惯自动写入 `{done:true, value:25}`（若已有值则累加），打卡页同步刷新

**AC-6.3 暂停恢复**
- **Given** 番茄钟运行中剩余 12:30
- **When** 用户点击暂停
- **Then** 倒计时停止，按钮变为"继续"；点击继续后从 12:30 继续而非重置为 25:00

### 3.7 数据导入导出

**AC-7.1 JSON 导出**
- **Given** 用户有 10 条习惯与 30 天打卡记录
- **When** 点击"导出 JSON"
- **Then** 生成包含 `habitsConfig` 与 `checkinRecords` 的 JSON 文件，文件名含日期（如 `lifestyle-backup-2026-07-18.json`），触发浏览器下载

**AC-7.2 CSV 导出**
- **Given** 用户有打卡数据
- **When** 点击"导出 CSV"
- **Then** 生成 CSV 文件，列为 `日期, 习惯名, 类型, 数值, 是否完成`，UTF-8 BOM 编码确保 Excel 中文不乱码

**AC-7.3 导入覆盖确认**
- **Given** 用户选择了一个有效 JSON 文件
- **When** 解析成功
- **Then** 弹出确认框"导入将覆盖当前数据，是否继续？"，确认后才执行覆盖并刷新 UI

**AC-7.4 异常文件**
- **Given** 用户选择了一个非法 JSON 文件
- **When** 尝试导入
- **Then** 提示"文件格式错误，导入失败"，现有数据不受影响

### 5.2 功能依赖链（文字描述）

1. **习惯打卡** → 依赖 `storage.js`（读写 habitsConfig / checkinRecords）、`data/habits.js`（习惯库定义）；
2. **数据统计** → 依赖 **习惯打卡** 产生的 `checkinRecords`，以及 `checkin.js` 的 `getStreak / getMaxStreak / getCompletionRate`；
3. **提醒系统** → 依赖 **习惯打卡** 的 `defaultReminder / intervalReminder` 配置；
   - `notification.js`（PWA 前台通知）与 `local-notify.js`（APK 原生通知）通过 `__PLATFORM__` 分化；
   - `push.js`（Web Push 后台）依赖 **Cloudflare Worker** 的 Cron + VAPID；
4. **番茄钟** → 依赖 **习惯打卡**（关联习惯后写回 checkinRecords）、`notification.js`（完成音效/提醒）；
5. **AI 顾问** → 依赖 **Cloudflare Worker**（ai-proxy.js 隐藏 Key）、`storage.js`（读取 ai_config）；可选依赖 **体质测评** 结果与 **打卡数据** 作为上下文；
6. **体质测评** → 独立问卷，但结果可被 **AI 顾问** 读取用于个性化建议；
7. **知识库** → 静态资源（`www/references/`），仅依赖 Service Worker 缓存；
8. **数据导入导出** → 依赖 `storage.js` 的 `habitsConfig / checkinRecords` 读写钩子；
9. **新手引导** → 依赖 localStorage `has_seen_guide` 标记，引导用户至打卡/管理/AI 各模块。

### 5.3 关键耦合点

- **`window.habitsConfig` / `window.checkinRecords`**：由 `storage.js` 通过 `Object.defineProperty` 定义为受控代理，所有模块统一读写入口，避免数据源分裂；
- **`_saveHooks`**：`saveConfig()` 后触发注册的钩子（如通知系统重新调度），是配置变更通知的核心机制；
- **`App.registerModule(name, layer, deps)`**：模块注册统一入口，用于按序加载与依赖声明。
