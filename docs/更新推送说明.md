# 应用内检查更新（方案 B）使用说明

已实现「打开 App 自动检查 + 设置页手动检查」的纯前端更新推送。
新版本仍由你手动出 APK 分发，但用户一打开 App 就能看到「发现新版本」弹窗并一键去下载。

---

## 一、已完成的代码改动
| 文件 | 改动 |
|------|------|
| `android/app/build.gradle` | 读取 `local.properties` 的 `UPDATE_JSON_URL`，并把本地 `versionCode`/`versionName` 注入 `app-config.js` |
| `scripts/build.mjs` | `modules.js` 打包列表新增 `modules/update.js` |
| `www/js/modules/update.js` | 新建：节流自动检查、版本对比、轻量更新弹窗 |
| `www/js/main.js` | 初始化后自动调用 `Update.check(false)`（每天最多一次） |
| `www/index.html` | 设置页新增「版本与更新」区块（显示当前版本 + 检查更新按钮） |
| `version.json` | 版本信息示例，可直接托管 |

---

## 二、版本信息 JSON 格式（托管在任意公开地址）
```json
{
  "versionCode": 2,
  "versionName": "1.1",
  "apkUrl": "https://你的托管/app-debug.apk",
  "whatsNew": [
    "修复习惯库搜索中文输入法被打断的问题",
    "新增应用内检查更新"
  ],
  "publishedAt": "2026-07-07"
}
```
- `versionCode` 必须是**整数且大于用户已装版本**，App 用它判断是否要更新。
- `apkUrl` 是新 APK 的下载地址（用户在弹窗点「立即更新」会打开它）。

## 三、把 version.json 托管出去（任选其一）
- **GitHub Pages（最简单，你已有仓库 `rui66648/lifestyle-assistant`）**
  把 `version.json` 推到仓库，仓库 Settings → Pages 开启，得到
  `https://rui66648.github.io/lifestyle-assistant/version.json`。
- 阿里云 OSS / 腾讯云 COS 静态网站。
- 你的 FC 函数直接返回这段 JSON（把 `apkUrl` 指向 OSS/COS 上的 APK）。

## 四、在 App 里配置更新地址
编辑 `android/local.properties`（此文件已被 gitignore，不会进仓库）：
```properties
UPDATE_JSON_URL=https://你的托管/version.json
```
> 改完 `local.properties` 后必须重新打包（`preBuild` 才会重新生成 `app-config.js`）。

## 五、发新版本的标准流程（务必遵守）
1. **改版本号**：`android/app/build.gradle` 里
   `versionCode 1 → 2`、`versionName "1.0" → "1.1"`。
   ⚠️ `versionCode` 必须递增，否则 Android 会以「版本不符」拒绝覆盖安装。
2. **重新出包**：
   ```powershell
   $env:JAVA_HOME="D:\Android\Android Studio\jbr"
   $env:ANDROID_HOME="C:\Users\瑞\AppData\Local\Android\Sdk"
   cd d:\AndroidStudioProjects\养生助手\lifestyle-assistant\android
   .\gradlew.bat assembleDebug
   ```
3. **上传 APK**：把新 `app-debug.apk` 传到 `version.json` 里 `apkUrl` 指向的地址。
4. **更新 version.json**：把 `versionCode`/`versionName`/`apkUrl`/`whatsNew` 改成新值并重新托管。
5. **用户侧**：打开 App（或设置页点「检查更新」）→ 弹「发现新版本 v1.1」→ 点「立即更新」去下载安装。

## 六、本地自测
1. 随便起个静态服务把 `version.json`（versionCode 设 2）放出去，或用上面的 GitHub Pages。
2. `local.properties` 填 `UPDATE_JSON_URL=...`，重新 `assembleDebug` 并安装。
3. 打开 App 应自动弹出更新提示（因 2 > 本地 1）。

> 当前已打好的包 `updateJsonUrl` 为空，点「检查更新」会提示「未配置更新地址」——属正常，配置后即生效。
