# 养生助手 FC 后端部署（AI 代理 + 用户认证 + 数据同步）

把后端部署到**国内**的阿里云函数计算，实现「App 零配置、Key 留在服务端、国内可直连」。

> 与 App 端 `ai.js` 和 `auth.js` 的接口完全兼容，**无需改 App 代码**。

---

## 一、创建阿里云 RDS MySQL（免费版）

1. 打开 [RDS 控制台](https://rds.console.aliyun.com)
2. **创建实例** → 选择 **MySQL** → **基础版**（免费试用）
3. 配置：
   - 数据库版本：MySQL 8.0
   - 地域：华东 2（上海）或离你最近的区域
   - 实例规格：免费试用（1核1G，50GB）
   - 网络类型：**专有网络（VPC）**
4. 创建完成后，记录：
   - **内网地址**（如 `rm-xxxx.mysql.rds.aliyuncs.com`）
   - **端口**（默认 3306）
   - **账号**（如 `admin`）
   - **密码**（你设置的）

### 配置安全组
1. RDS 控制台 → 实例 → **安全组配置**
2. 添加安全组规则：允许 **0.0.0.0/0** 访问（或仅允许函数计算 VPC 访问）

### 初始化数据库
1. RDS 控制台 → 实例 → **登录数据库**
2. 执行 `init-db.sql` 脚本（文件在本目录）：

```sql
CREATE DATABASE IF NOT EXISTS lifestyle_assistant DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
USE lifestyle_assistant;

CREATE TABLE IF NOT EXISTS users (
  id          VARCHAR(36) PRIMARY KEY,
  phone       VARCHAR(11) UNIQUE NOT NULL,
  email       VARCHAR(255) UNIQUE,
  password    VARCHAR(128) NOT NULL,
  salt        VARCHAR(64) NOT NULL,
  nickname    VARCHAR(20) DEFAULT '',
  created_at  BIGINT NOT NULL,
  updated_at  BIGINT NOT NULL,
  INDEX idx_users_phone (phone),
  INDEX idx_users_email (email)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

CREATE TABLE IF NOT EXISTS refresh_tokens (
  token       VARCHAR(64) PRIMARY KEY,
  user_id     VARCHAR(36) NOT NULL,
  device_id   VARCHAR(64),
  expires_at  BIGINT NOT NULL,
  created_at  BIGINT NOT NULL,
  INDEX idx_refresh_tokens_user (user_id),
  INDEX idx_refresh_tokens_expires (expires_at),
  FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

CREATE TABLE IF NOT EXISTS user_data (
  user_id     VARCHAR(36) NOT NULL,
  data_key    VARCHAR(64) NOT NULL,
  data_value  TEXT NOT NULL,
  updated_at  BIGINT NOT NULL,
  PRIMARY KEY (user_id, data_key),
  INDEX idx_user_data_user (user_id),
  FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

## 二、部署函数计算

### 方式 A：函数计算控制台（推荐，无需 CLI）

1. 打开 [函数计算控制台](https://fc.console.aliyun.com)
2. 左侧 **服务与函数** → 创建服务，名称 `lifestyle-assistant`
3. 进入服务 → **创建函数**：
   - 创建方式：**Web 函数**
   - 运行环境：**Node.js 20**
   - 函数名称：`lifestyle-api`
   - 上传 `serverless/aliyun-fc/index.js`（或在线编辑器粘贴本文件内容）
   - 触发器：**HTTP 触发器**，认证方式选 **anonymous（免鉴权）**
   - 请求方法勾选 `GET / POST / PUT / OPTIONS`
4. 函数配置 → **环境变量** 添加：

| 变量名 | 值 | 说明 |
|--------|-----|------|
| `QWEN_API_KEY` | `sk-你的阿里百炼Key` | AI 代理 |
| `JWT_SECRET` | `openssl rand -hex 32` | JWT 签名密钥 |
| `DB_HOST` | `rm-xxxx.mysql.rds.aliyuncs.com` | RDS 内网地址 |
| `DB_PORT` | `3306` | MySQL 端口 |
| `DB_USER` | `admin` | 数据库账号 |
| `DB_PASSWORD` | `你的密码` | 数据库密码 |
| `DB_NAME` | `lifestyle_assistant` | 数据库名 |

5. 记录 HTTP 触发器给出的**公网地址**，形如：
   ```
   https://xxx.cn-shanghai.fcapp.run
   ```

6. 本地改 `android/local.properties`：
   ```properties
   CLOUD_AI_URL=https://xxx.cn-shanghai.fcapp.run
   CLOUD_AI_KEY=          # 留空！走服务端代理
   CLOUD_AI_MODEL=qwen-max
   ```

7. 重新构建：
   ```bash
   npx cap sync
   npx cap build android
   ```

### 验证函数是否存活
```bash
# GET 请求检查服务状态
curl https://xxx.cn-shanghai.fcapp.run

# 应返回：{"ok":true,"service":"lifestyle-fc","ai":true,"auth":true}
```

### 测试认证接口

## 四、接口说明

### AI 代理（POST /）
与之前完全兼容，App 无需改动。

### 用户认证
| 端点 | 方法 | 说明 |
|------|------|------|
| `/auth/register` | POST | 注册（phone, password, nickname） |
| `/auth/login` | POST | 登录（phone, password） |
| `/auth/refresh` | POST | 刷新 token（refreshToken） |
| `/auth/logout` | POST | 登出 |
| `/auth/me` | GET | 获取用户信息 |
| `/auth/me` | PUT | 更新昵称 |

### 数据同步
| 端点 | 方法 | 说明 |
|------|------|------|
| `/sync/upload` | POST | 上传数据（habits_config, checkin_records, constitution_result） |
| `/sync/download` | GET | 下载数据 |

## 六、注意事项

- **安全性**：HTTP 触发器为匿名公开，任何人拿到地址都能调用。已内置每 IP 每小时 50 次限流；不要公开泄露你的 FC 地址即可。
- **接口兼容**：与 Cloudflare Worker 方案接口一致，可随时切换。
- **数据库连接池**：使用 mysql2/promise 连接池，复用连接提高性能。
- **JWT Token**：accessToken 有效期 15 分钟，refreshToken 有效期 30 天。
