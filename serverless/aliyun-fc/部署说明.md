# 阿里云函数计算（FC）版 AI 代理部署

把 AI 代理部署到**国内**的阿里云函数计算，替代被墙的 Cloudflare `*.workers.dev`，
实现「App 零配置、Key 留在服务端、国内可直连」。

> 与 App 端 `ai.js` 的「Worker 代理模式」接口完全兼容，**无需改 App 代码**。

---

## 方式 A：函数计算控制台（推荐，无需 CLI）

1. 打开 [函数计算控制台](https://fc.console.aliyun.com)，确认已开通（华东2 上海 `cn-shanghai` 等）。
2. 左侧 **服务与函数** → 创建服务，名称 `lifestyle-assistant`。
3. 进入服务 → **创建函数**：
   - 创建方式：**Web 函数**（或「使用自定义运行时创建」→ HTTP 触发器）
   - 运行环境：**Node.js 20**
   - 函数名称：`ai-proxy`
   - 上传 `serverless/aliyun-fc/index.js`（或在线编辑器粘贴本文件内容）
   - 触发器：**HTTP 触发器**，认证方式选 **anonymous（免鉴权，公开）**
   - 请求方法勾选 `GET / POST / OPTIONS`
4. 函数配置 → **环境变量** 添加：
   ```
   QWEN_API_KEY = sk-你的阿里百炼Key
   ```
   （设置 → 环境变量 → 新增，保存）
5. 记录 HTTP 触发器给出的**公网地址**，形如：
   ```
   https://xxx.cn-shanghai.fcapp.run
   ```
6. 本地改 `android/local.properties`：
   ```properties
   CLOUD_AI_URL=https://xxx.cn-shanghai.fcapp.run
   CLOUD_AI_KEY=          # 留空！走服务端代理，不需要客户端 Key
   CLOUD_AI_MODEL=qwen-max
   ```
7. 重新构建：
   ```bash
   npx cap sync
   npx cap build android
   ```
8. 装到手机 → 设置显示「已配置 · qwen-max」，AI 顾问直接可用。

### 验证函数是否存活
浏览器/终端访问你的 FC 地址（GET）：
```
https://xxx.cn-shanghai.fcapp.run
```
应返回 `{"ok":true,"service":"lifestyle-ai-proxy-fc","dashscope":true}`（`dashscope:true` 表示 Key 已配置）。

## 方式 B：Serverless Devs（s 工具，适合自动化）

```bash
npm install -g @serverless-devs/s
s config add            # 用阿里云 AccessKey 登录
export QWEN_API_KEY=sk-你的Key
cd fc
s deploy               # 按 s.yaml 创建服务/函数/HTTP 触发器
```
部署完成后终端会打印公网地址，按方式 A 第 6 步填进 `local.properties`。

## 本地联调（未部署 FC 前快速验证）

不想先部署也能端到端验证链路：用 `fc/local-dev.cjs` 在本地起一个与 FC 完全一致的代理，
把 App 的 `CLOUD_AI_URL` 指向本机即可。

```bash
# 进入 fc 目录，设置你的 Key（仅本机内存，不入仓库）
$env:QWEN_API_KEY="sk-你的Key"
node local-dev.cjs
```

默认监听 `http://localhost:3000`，控制台会提示 Android 模拟器访问 `http://10.0.2.2:3000`。
接着在 `android/local.properties` 填：

```properties
CLOUD_AI_URL=http://10.0.2.2:3000      # 模拟器；真机用电脑局域网 IP
CLOUD_AI_KEY=
CLOUD_AI_MODEL=qwen-max
```

```bash
npx cap sync && npx cap build android
```

装到模拟器/手机 → 设置显示「已配置 · qwen-max」，AI 顾问应正常回复，说明整套
`local.properties → Gradle → app-config.js → ai.js` 注入链路已打通。验证通过后，
再把 `CLOUD_AI_URL` 换成正式 FC 公网地址即可。

> 注意：本地联调仅用于开发验证；正式发布请用 FC 公网地址，而非本机 `localhost`。

## 说明与提醒

- **接口兼容**：App 端 `ai.js` 的 Worker 模式直接向 `{CLOUD_AI_URL}` 发 `POST`，
  本函数会补上 `SYSTEM_PROMPT` 并透传 OpenAI 格式响应，因此 App 无需任何改动。
- **安全性**：HTTP 触发器为匿名公开，任何人拿到地址都能调用。已内置每 IP 每小时 50 次限流；
  不要公开泄露你的 FC 地址即可。若需更强控制，可在函数里加 `PROXY_TOKEN` 校验（需同步改 `ai.js`）。
- **费用**：FC 有每月免费额度（前 100 万次调用/月免费档），自用几乎零成本。
- **与 Cloudflare Worker 方案的关系**：二者接口一致，可随时切换。
  FC 在国内可达、无域名备案烦恼，是当前的推荐方案。
