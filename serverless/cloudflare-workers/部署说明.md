# Cloudflare Worker 部署说明（AI 代理 + MCP 代理 + 后台推送）

> **🎉 好消息：推送服务已内置！**
> 
> 用户使用时**无需部署**，直接打开应用 → 点击「一键开启推送」即可。
> 以下文档仅适用于开发者自行部署私有实例。

---

## 🏗️ 开发者部署（可选）

### 一键部署

```bash
cd workers/
bash deploy.sh
```

### 手动部署

1. 设置 Secret：`VAPID_PRIVATE_KEY`、`QWEN_API_KEY`
2. 创建 KV 命名空间 `PUSH_KV` 并绑定
3. 设置 Cron 触发器：`* * * * *`
4. 部署 Worker

### 启用 MCP 代理（可选）

Worker 现在支持代理 ModelScope MCP 服务，为 AI 养生顾问提供：

- **HowToCook-MCP**：菜谱查询、膳食规划
- **kb-mcp-server**：养生知识库语义检索
- **Strava-MCP**：运动数据获取（跑步、骑行、心率等）

配置方式：

1. 在 Cloudflare Worker 设置中添加 Secret：`MODELSCOPE_API_TOKEN`
   - 获取地址：https://www.modelscope.cn/my/myaccesstoken
2. 或在 .env 中直接配置硬编码 URL：
   - `HOWTOCOOK_MCP_URL`
   - `KNOWLEDGE_MCP_URL`
   - `STRAVA_MCP_URL`（Strava-MCP 需要额外完成 Strava 认证）
3. 前端 AI 设置中填写 Worker URL 和 API Key 即可使用

MCP 路由：

- `POST /mcp/howtocook` - 调用 HowToCook-MCP
- `POST /mcp/knowledge` - 调用 kb-mcp-server
- `POST /mcp/strava` - 调用 Strava-MCP（需完成 Strava 认证）

#### Strava-MCP 额外配置

Strava-MCP 需要完成 Strava API 认证：

1. 在 [Strava API 设置](https://www.strava.com/settings/api) 创建应用
2. 配置回调域名和获取 Client ID、Client Secret
3. 运行认证脚本完成 OAuth 授权
4. 将生成的 `STRAVA_ACCESS_TOKEN` 和 `STRAVA_REFRESH_TOKEN` 配置到 Strava-MCP 服务端

## 📱 用户使用

1. 打开应用 → [https://rui66648.github.io/lifestyle-assistant/](https://rui66648.github.io/lifestyle-assistant/)
2. 点击底栏「我的」→ 设置 → 🔔 **后台推送提醒**
3. 点击「🔔 一键开启推送」→ 允许通知权限
4. 搞定！习惯提醒会通过云端推送 ✅

> 📱 **iOS 用户**：需先 Safari「分享 → 添加到主屏幕」，从桌面图标打开后再开启。
