# 绑定自定义域名（免费 + 国内直连）

目标：让 Worker 用你自己的普通域名（如 `api.你的名.is-a-dev`）访问，绕开被墙的 `*.workers.dev`。

## 第 1 步：Cloudflare 拿到 NS
1. 登录 [dash.cloudflare.com](https://dash.cloudflare.com) → **Add a Site** → 输入 `你的名.is-a-dev`（换成你想要的子域）。
2. Cloudflare 会显示两个 NS，例如 `xxx.ns.cloudflare.com` / `yyy.ns.cloudflare.com`。
3. **复制这两个 NS**（先别急着“继续”，只是看一眼拿到值）。

## 第 2 步：提交 is-a.dev 申请（把 NS 交给 is-a.dev）
1. 打开 <https://github.com/is-a-dev/register> → 点 **Fork**。
2. 在仓库 `domains/` 目录下新建文件 `你的名.json`，内容用本目录的 `is-a-dev-domain.json` 模板：
   - 把 `NS` 两行换成第 1 步拿到的 Cloudflare NS。
   - 把 `username` / `email` 换成你自己的。
3. 提交 **Pull Request**，等审核（几分钟~几小时）。
4. 审核通过后，`你的名.is-a-dev` 的 NS 就指向了 Cloudflare。

## 第 3 步：Cloudflare 完成接入
1. 回到 Cloudflare **Add a Site** 那个流程，点继续 → 选 **Free** 计划。
2. 等待 Cloudflare 检测到 NS 生效（几分钟~24 小时），状态变 **Active**。

## 第 4 步：绑定到 Worker
1. 编辑本目录 `wrangler.toml`，把最后一行：
   ```toml
   name = "REPLACE_WITH_YOUR_CUSTOM_DOMAIN"
   ```
   改成：
   ```toml
   name = "api.你的名.is-a-dev"
   ```
   （建议用 `api.` 子域，避免和将来其他用途冲突）
2. 确保已配好 Secret：
   ```bash
   cd workers
   npx wrangler secret put QWEN_API_KEY
   npx wrangler secret put VAPID_PRIVATE_KEY
   ```
3. 部署（会自动把 Worker 绑到该域名并签发免费 SSL）：
   ```bash
   npx wrangler deploy
   ```

## 第 5 步：填入 App
把最终地址 `https://api.你的名.is-a-dev` 发给助手，他会写入：
- `www/js/modules/ai.js` 的 `DEFAULT_WORKER_URL`
- `www/js/modules/push.js` 的 `DEFAULT_WORKER_URL`

写入后运行 `npm run build` 重建 bundle，AI 顾问 + Web 推送即全部内置、国内可直连。

---

## 备选免费域名
- **`eu.org`**：<https://www.eu.org> 申请 `你的名.eu.org`，审核 1~3 天，同样把 NS 指向 Cloudflare 后走第 3~5 步。
- **付费 `.com`**：约 ¥50~70/年，最稳，流程相同。
