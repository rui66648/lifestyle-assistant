## 使用说明

### 提示词结构规范

本手册中每个提示词遵循以下结构：

```
角色设定 -> 上下文描述 -> 具体约束 -> 分步执行 -> 验证标准
```

1. **角色设定**：为 AI 指定一个专业身份（如产品经理、前端工程师）
2. **上下文描述**：提供项目背景、技术栈、当前阶段信息
3. **具体约束**：列出技术约束、业务规则、代码规范
4. **分步执行**：用编号列表明确任务步骤
5. **验证标准**：定义输出质量检查清单

### 使用方法

- 将提示词整段复制后粘贴给 AI 编码助手
- 根据实际开发进度替换 `[方括号]` 中的变量内容
- 每个阶段可独立使用，也可串联形成完整开发流程
- 建议配合附录中的通用辅助提示词使用

```

- 饮食营养（diet）：吃早餐、均衡饮食、每日饮水等 10 项
- 学习成长（study）：阅读、背单词、练字等 10 项
- 睡眠作息（sleep）：早起、早睡、午休等 5 项
- 心灵修养（mind）：冥想、写日记、情绪记录等 7 项
- 五劳五色防护（protect）：闭目养神、起身活动、五色食物等 13 项

- 居家生活（home）：整理房间、照料植物等

每个习惯的数据结构为：
{
  id: string,           // 唯一标识
  name: string,         // 习惯名称
  icon: string,         // emoji 图标
  category: string,     // 分类
  timePeriod: string,   // 建议时段（morning/forenoon/afternoon/evening）
  type: string,         // 打卡类型（boolean/timer/count/water/select）
  unit: string,         // 单位
  tip: string,          // 养生小贴士

  intervalReminder: {}  // 间隔提醒（可选）

2. 每个习惯的 tip 字段需引用具体典籍或研究

【分步执行】
1. 审视现有习惯库，找出缺失的养生维度
2. 为每个缺失维度设计 3-5 个新习惯

6. 输出为可直接粘贴到 habits.js 的代码格式

【验证标准】
- [ ] 新习惯总数达到 120+ 项
- [ ] 每个 tip 有典籍出处或科学依据
- [ ] 数据结构与现有格式完全一致
- [ ] 推荐组合至少 5 套

### 提示词 2.1 -- 整体架构设计

```
【角色设定】
你是一位精通 PWA 和混合应用架构的高级前端架构师，对 Capacitor、Service Worker、离线优先架构有丰富经验。

项目「生活习惯小助手」采用以下技术架构：

前端层：
  - 纯 Vanilla JavaScript（无框架），模块化组织
  - 模块注册系统：App.registerModule(name, layer, factory)
  - 四层架构：data（数据定义）-> core（工具/存储）-> modules（业务逻辑）-> ui（渲染/交互）
  - 构建系统：scripts/build.mjs 将源文件按序拼接为 bundle（data.js/modules.js/ui.js）
  - 懒加载：www/js/lazy.js 实现按需加载模块（AI、番茄钟、海报等）
  - Service Worker：sw.js 实现离线缓存（Cache First 策略）

跨平台层：
  - Capacitor 8：将 PWA 包装为 Android 原生应用
  - 配置：capacitor.config.json（appId: com.rui66648.lifestyle）
  - 本地通知：@capacitor/local-notifications
  - 平台适配：www/js/core/platform.js 区分 Web/Android 环境

数据层：

  - CSV 导出：用于数据分析

【具体约束】

4. 兼容 Android 5.0+（API 21）和现代浏览器

【分步执行】
1. 绘制整体架构图（用文字/ASCII 描述层次关系）
2. 分析当前模块注册系统，提出优化建议
3. 设计数据流：用户操作 -> 模块处理 -> 存储更新 -> UI 刷新

【验证标准】
- [ ] 架构图清晰展示前端/跨平台/后端/数据四层关系

### B.2 -- 关键数据流

```
用户操作 -> 事件处理（ui/events.js）
         -> 模块方法（modules/*.js）
         -> 数据读写（core/storage.js -> localStorage）
         -> UI 刷新（ui/render.js -> DOM 更新）
         -> 动画反馈（CSS transitions/animations）
```

### B.3 -- 模块注册清单

```
data 层：constants -> content -> habits -> packs -> sports
core 层：utils -> storage
modules 层：checkin -> habit -> stats -> water -> diet -> sports -> pomodoro -> ai -> constitution -> poster -> notification -> guide -> update
ui 层：render -> panels -> components -> events
```
