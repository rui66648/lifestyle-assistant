# AI 养生顾问部署说明

## 概述

AI 养生顾问功能通过 Cloudflare Workers 代理 + 通义千问 API 实现，具备以下特性：
- 🔒 **安全**：API Key 隐藏在 Worker 环境变量中，不暴露在前端
- 🚀 **限流**：每 IP 每小时最多 50 次请求，防止滥用
- 📝 **对话记忆**：对话记录保存在本地 localStorage
- ✅ **XSS 防护**：使用 textContent 防止恶意脚本注入

## 架构图

```
用户浏览器 → Cloudflare Workers (ai-proxy.js) → 阿里百炼 API
                    ↓
              隐藏 API Key
              请求限流
              错误处理
```

## 部署步骤

### 1. 注册阿里百炼获取 API Key

1. 访问 https://bailian.console.aliyun.com
2. 注册/登录阿里云账号
3. 进入"百炼大模型"控制台
4. 点击右上角头像 → API Key 管理
5. 创建新的 API Key，复制保存

> 新用户有免费额度，足够日常使用。

### 2. 注册 Cloudflare 账号

1. 访问 https://dash.cloudflare.com
2. 注册账号（可用邮箱直接注册）
3. 进入 Workers & Pages 控制台

### 3. 创建 Worker

1. 点击"创建服务"或"Create a Service"
2. 服务名称填写 `ai-proxy`（或任意名称）
3. 点击"创建服务"

### 4. 编辑 Worker 代码

1. 点击"快速编辑"或"Quick Edit"
2. 删除默认代码，粘贴 `workers/ai-proxy.js` 中的全部内容
3. 点击"保存并部署"

### 5. 配置环境变量

1. 返回 Worker 主页，点击"设置" → "变量"
2. 添加环境变量：

| 变量名 | 值 | 说明 |
|:---|:---|:---|
| `QWEN_API_KEY` | 你的阿里百炼 API Key | 必填，API 认证密钥 |
| `RATE_LIMIT` | `50` | 可选，每小时最大请求数，默认 50 |

3. 点击"保存并部署"

### 6. 获取 Worker URL

1. 在 Worker 主页，找到触发器（Triggers）
2. 默认会有一个 `*.workers.dev` 的 URL
3. 复制这个 URL，例如：`https://ai-proxy.yourname.workers.dev`

### 7. 配置前端

配置已迁移到应用内设置面板，无需修改代码：

1. 打开应用（GitHub Pages 地址）
2. 点击底部导航的「我的」页面
3. 找到「AI 设置」入口，点击进入设置面板
4. 填写：
   - **Worker URL**：粘贴第 6 步获取的 Worker URL
   - **API Key**：粘贴第 1 步获取的阿里百炼 API Key
   - **模型**：选择 `qwen-turbo`（默认，轻量快速）
5. 点击「保存配置」

> 配置信息保存在浏览器 localStorage 中，不会上传到任何服务器。

### 8. 验证

1. 打开应用，点击右下角的 🤖 浮动按钮
2. 输入"失眠怎么调理？"
3. 如果看到 AI 回复，说明部署成功

## 功能说明

### 请求限流

- 每 IP 每小时最多 50 次请求
- 超限后会返回 429 错误，提示用户等待
- 响应头包含限流信息：
  - `X-RateLimit-Limit`：限制次数
  - `X-RateLimit-Remaining`：剩余次数
  - `X-RateLimit-Reset`：重置时间（秒）

### 对话历史

- 对话记录保存在浏览器 localStorage
- 刷新页面后历史记录保留
- 点击"清空对话"可清除历史

### 安全特性

- ✅ API Key 隐藏在 Worker 环境变量
- ✅ XSS 防护：使用 textContent 转义内容
- ✅ 输入长度限制：单条消息最多 2000 字符
- ✅ 错误处理：友好的错误提示

## 故障排查

| 问题 | 原因 | 解决 |
|:---|:---|:---|
| 显示配置说明 | API Key 未填写 | 在应用设置面板中填写 API Key |
| 显示"请稍后再试" | 请求超限 | 等待一小时后再试 |
| 网络错误 | Worker URL 错误 | 检查 URL 是否完整 |
| API 认证失败 | API Key 错误 | 检查环境变量 QWEN_API_KEY |
| 回复为空 | API 额度用完 | 检查阿里百炼控制台额度 |

## 升级建议

### 阶段一（当前）：单 Worker + 单 API

适合用户量 < 1000 人

### 阶段二：多 API 备份

同时接入 2-3 家 AI 提供商（阿里百炼 + 智谱 + 月之暗面），一家挂了自动切换

### 阶段三：KV 精确限流

使用 Cloudflare KV 存储限流数据，实现跨实例精确限流
